# methodology.md

> 산업 분석·종목 평가 시스템의 룰북. 모든 채점·갱신·이벤트 처리는 이 문서를 기준으로 한다.
> 임계치·룰 변경은 **분기 첫 주에만**. 채점 중간에 변경 금지. 변경 시 본 문서 하단 업데이트 로그에 사유 기록.

---

## 1. 시스템 목적과 운영 원칙

### 1.1 목적

**효율 도구**. 사용자(이하 "본인")의 산업 분석·종목 평가 의사결정을 위한 정보의 효율적 취합·구조화·일관된 채점.

매수·매도 의사결정, 자본 배분, 매수 시점·가격은 **시스템이 결정하지 않는다**. 시스템은 본인이 의사결정 시점에 일관된 인풋을 받을 수 있도록 정보를 정리한다.

### 1.1.1 시스템 적용 범위 (Scope)

본 시스템은 **변화(change)에 베팅하는 투자 전략용**이다. 다음 영역은 시스템 범위 밖:

- 안정적 배당주 (장기 보유, 안정 배당) — Moat가 안정적인 게 미덕인 영역
- 전통적 블루칩 (대형 안정 기업, 변화 적음)
- 단기 트레이딩·차트 베이스
- 인덱스·ETF 패시브 투자
- 채권·원자재·FX

본 시스템의 채점 (특히 Moat 잣대) 은 "변화가 크면 매력적"으로 설계됨. 안정성을 추구하는 투자에는 부적합. 본인이 이 시스템 외 별도 안정 자산을 운용하는 것은 본인 판단.

### 1.2 평가 기준

시스템의 좋고 나쁨은 다음으로 평가한다:
- 본인 의사결정 시간 절약 (같은 결정을 손으로 했을 때 대비)
- 채점의 일관성 (같은 영역을 6개월 간격으로 채점해도 비슷한 결과)
- 노이즈 차단 효율 (다이제스트에 진짜 봐야 할 것만 올라오는가)
- 본인 일정 종속성 없음 (며칠~몇 주 부재 시에도 정보 유실 없음)

**수익률은 평가 기준이 아니다.** 수익률은 본인 매매 결정의 결과이고, 시스템 품질의 직접 측정이 될 수 없다.

### 1.3 본인 일정 독립성

본인이 다이제스트·이벤트·재평가를 며칠~몇 주 놓쳐도 시스템은 정보를 유실하지 않는다:

- **다이제스트 아카이브**: 모든 다이제스트는 `digests/YYYY-MM-DD.md` 로 영구 보존. 본인이 N일 만에 복귀 시 "지난 N일 요약본" 자동 생성.
- **이벤트 큐**: 본인이 응답 안 한 우선순위 1~2 이벤트는 `pending-review/` 로 자동 이동. 일정 기간 후에도 삭제되지 않음. 본인이 복귀 시 일괄 검토.
- **분기 재평가 보류**: 본인이 분기 첫 주에 검토 불가 시 Claude가 초안만 작성하고 `pending-review/` 로 보류. 다음 분기 전에만 처리되면 됨.
- **임계치 변경 일정 종속 없음**: 분기 첫 주 일정 놓치면 다음 분기로 자동 이월.

### 1.4 시스템 vs 본인 역할 분담

자동화 수준: **Level 2 + 일부 Level 3** (분석·채점 보조, 의사결정 본인).

| Task | 시스템 (Claude) | 본인 |
|------|----------------|------|
| 메가 체인지 후보 발굴 | 분기마다 초안 갱신 | 최종 리스트 승인 |
| 4가지 잣대 채점 | 초안 (수치·confidence 포함) | 검토·수정·승인 |
| Tier 승강 | 룰에 따라 제안 | 승인 |
| 밸류체인 매핑 | 초안 | 검토·보완 |
| 종목 후보 매핑 | 후보 종목 리스트 | 워치리스트 등재 결정 |
| Valuation Gate 채점 | 초안 (4 게이트) | 검토·승인 |
| 일간 다이제스트 | 완전 자동 생성 | 검토 (선택적) |
| 트리거 이벤트 알림 | 우선순위 1~2 자동 푸시 | 액션 결정 |
| 매크로 트래커 갱신 | 자동 수집·요약 | 주간 검토 |
| VC 펀딩·CVC 신호 | 자동 수집·분류 | 임팩트 평가 |
| 스타트업↔상장사 매핑 | 초안 | 검토·보완 |
| Thesis 문서 작성 | 템플릿 따라 초안 | 검토·승인·서명 |
| Thesis 업데이트 | diff 제안 | 승인 후 반영 |
| 분기 재평가 | 모든 영역 재채점 초안 | 1~2시간 들여 최종 결정 |
| **매수·매도** | — | **본인 단독** |
| **자본 배분** | — | **본인 단독** |
| **매수 시점·가격** | — | **본인 단독** |

### 1.5 컴플라이언스 원칙

- 본인이 VC 업무에서 알게 된 미공개 정보로 *연관 상장사 주식*을 매매하는 것은 자제. (예: 본인이 검토한 비상장 A사의 비공개 매출 정보 → A의 주요 고객 상장 B사 매매)
- 시스템에는 **공개 정보**만 입력. 본인 회사 IC 자료, 검토 중인 회사의 비공개 자료는 시스템에 넣지 않음.
- 산업 트렌드·집계 신호 (예: "이 영역에 시리즈 C 라운드가 분기 5건 이상 일어나고 있다") 는 활용 가능.
- 본인 회사 컴플라이언스 룰을 시스템보다 우선 적용.
- 위반 의심 시: 본인이 자체 판단. 시스템은 본인이 입력한 정보 그대로 처리.

## 2. 분석 프레임워크

### 2.1 5-layer 아키텍처

```
L0. methodology.md (이 문서) — 모든 채점·갱신의 룰북
   ↓
L1A. Global mega change map     L1B. Korea-specific tracks
     (4-lens 채점 + 시차)             (4-lens + 한국 가중치)
   ↓                                ↓
L2. Startup bridges (스타트업 ↔ 상장사 매핑, 양쪽에 시그널 공급)
L3. Macro trackers (KR/US/지정학/VC 펀딩, 양쪽에 외생 변수 공급)
   ↓
L4. Watchlist (종목 + Valuation Gate)
   ↓
L5. Output layer (일간/주간/분기 다이제스트, 액션 아이템)
```

### 2.2 두 트랙: 글로벌과 한국

**L1A — Global mega change map**
- 글로벌 메가 체인지 분석. 잣대는 글로벌 기준.
- 미국 발 (대부분), 일부 중국·유럽 발 변화.
- 종목 매핑은 미국 + 한국 transmission 종목 동시 (한국 종목이 글로벌 변화에 노출돼 있으면 여기에 매핑).

**L1B — Korea-specific tracks**
- 한국 발 메가 체인지·사이클·정책 변화.
- 같은 4가지 잣대를 쓰되, **한국 컨텍스트 임계치**로 조정 (2.4 참조).
- 추가 잣대: Geopolitical alignment, Policy tailwind (2.5 참조).
- 대표 영역: K-방산, K-원전 수출, 조선·해운 사이클, 거버넌스 개선 (밸류업), 정책 사이클.

### 2.3 분석 순서 (모든 영역에 공통 적용)

영역이 어떤 트랙이든 다음 5단계로 분석한다.

**Step 1. 산업 메가트렌드 / 사이클**
- 수요 측: 무엇이 수요를 만드는가, 구조적인가 사이클인가
- 공급 측: CapEx 사이클 위치, 공급 따라오는가
- 정책·지정학: 정책·규제·보조금·수출통제

**Step 2. 밸류체인 매핑**
- 업스트림 (소재·장비) → 미드스트림 (부품·모듈) → 다운스트림 (완제품·서비스)
- 레이어별: 마진 수준 / 경쟁 강도 / 중국 침투 여부 / 진입장벽 종류

**Step 3. 병목 식별**
- 현재 병목: 지금 누가 부족한 것을 쥐고 있는가
- 다음 병목 (★ 투자 핵심): 현재 병목 해소 후 어디가 막힐 것인가
- 컨센서스는 보통 현재 병목까지만 본다. 다음 병목에 선제 포지셔닝.

**Step 4. 자금 흐름 시그널**
- 관련 ETF AUM 변화
- 외국인·기관 수급 (한국)
- 13F 변화, 옵션 포지션 (해외)
- CapEx 가이던스, 신규 수주 공시
- VC 펀딩 추이 (L2 신호)

**Step 5. 종목 매핑**
- Step 2 밸류체인의 각 위치에 글로벌 + 한국 종목 핀
- 각 종목별: 시장 점유율, 매출 비중, 트리거 이벤트, 기각 조건
- Valuation Gate 통과 여부 (별도 채점, 섹션 5에서 정의)

### 2.4 4가지 잣대 — 모든 영역 채점의 기준

영역의 매력도는 다음 4가지 잣대로 채점한다. 각각 0~5점, 합산 0~20점.

#### 잣대 1: TAM (시장 규모의 자릿수 변화)

5년 시계 기준 시장 규모 변화. **절대값**과 **배수** 둘 다 본다.

| 점수 | 정량 기준 | 카테고리 |
|------|---------|---------|
| 5 | 5년 내 ×10 이상 + 절대값 100B$+ 도달 | Mega-transformative |
| 4 | 5년 내 ×3~10 + 절대값 50B$+ | Major expansion |
| 3 | 5년 내 ×2~3 + 절대값 20B$+ | Significant growth |
| 2 | 5년 내 ×1.5~2 | Above-trend growth |
| 1 | 5년 내 ×1~1.5 | Normal growth |
| 0 | 정체 또는 축소 | No change / decline |

**한국 트랙 조정**: 한국 트랙에서는 글로벌 절대값 기준은 완화 가능. 한국 기업의 글로벌 시장 확보 가능 매출 기준으로 대체. (예: K-방산의 한국 기업 수출액 추정.)

#### 잣대 2: Moat (해자 재편 + 비가역성)

기존 강자의 해자가 무력화되거나 새 해자가 형성되는 정도 **+** 그 변화가 되돌릴 수 없는 정도. 5년 시계.

> **통합 사유**: Tech 잣대(비가역성)와 Moat 잣대(해자 재편)는 본질적으로 같은 것을 다른 각도에서 본 것. "기존 강자가 도태된다"와 "되돌릴 수 없는 변화"는 거의 같은 현상. 분리하면 같은 점에 두 번 점수가 매겨져 채점 왜곡 발생. 통합하여 단일 잣대로 처리.

| 점수 | 정량 기준 |
|------|---------|
| 5 | 기존 강자 1~3개가 5년 내 도태 또는 신규 진입자에 시장 50%+ 빼앗김 + 물리적·이론적 비가역 (되돌리려면 비용 폭발 또는 불가능) |
| 4 | 기존 강자 마진 구조 영구 재편 (마진율 절반 이하로) + 5년 내 기존 기술/지위 30%+ 대체 확실 |
| 3 | 새 카테고리 형성 (기존 가치사슬 위에 신규 레이어 추가) + 일부 비가역 진행 |
| 2 | 점진적 점유율 이동 (10~30%) + 되돌릴 수 있으나 비효율적 |
| 1 | 미미한 변화. 비가역성 약함. |
| 0 | 기존 강자 안정적 유지. 변화 없음. |

**중요**: 본 시스템은 **변화 베팅용**. 기존 강자가 안정적인 영역 (Moat 0~1점) 은 매력적이지 않다. 안정적 배당주·블루칩 투자는 본 시스템 범위 밖.

#### 잣대 3: Capital (자본 집중)

빅테크·정부·VC 자본 유입의 연간 규모. 글로벌 합계 기준.

| 점수 | 정량 기준 |
|------|---------|
| 5 | 연 500B$+ 투입 (예: 빅테크 4사 AI CapEx 합계 수준) |
| 4 | 연 100~500B$ |
| 3 | 연 30~100B$ |
| 2 | 연 10~30B$ |
| 1 | 연 1~10B$ |
| 0 | 의미 있는 자본 유입 없음 |

**합산 대상**: 빅테크 CapEx + 정부 정책 자금 + VC 펀딩 + 사모펀드 인프라 투자. 중복 카운트 주의.

**한국 트랙 조정**: 한국 정부 정책 자금 + 한국 기업 CapEx + 한국 CVC 합계. 임계치는 글로벌 대비 1/10 수준으로 조정 가능 (예: 5점 = 연 50B$+).

#### ~~잣대 4: Tech~~ (폐기됨)

> **2026-05-23 변경**: Tech 잣대는 "비가역성"과 "성숙도"라는 서로 다른 두 차원을 한 점수에 섞고 있어 채점 왜곡 발생. 비가역성은 Moat 잣대에 흡수, 성숙도는 별도 잣대가 아니라 A·B·C 시차의 B 지표(실물 변화 진행도)에서 이미 측정 중이므로 폐기. 결과적으로 글로벌 트랙은 3잣대, 한국 트랙은 5잣대로 축소.

#### 잣대 4: Geopolitical Alignment (L1B 전용)

한국이 미·중 경쟁 구도에서 이 산업에서 유리한 위치에 있는가.

| 점수 | 기준 |
|------|------|
| 5 | 미국이 한국 의존도 큰 영역 + 중국 대체 어려움 (예: 반도체 메모리, 일부 디스플레이) |
| 4 | 미국 동맹 구조 안에서 한국이 핵심 공급사 |
| 3 | 양쪽 다 거래 가능, 한국이 중립적 위치 |
| 2 | 미국·중국 양쪽에서 견제받는 위치 |
| 1 | 한국의 시장 접근 제한 |
| 0 | 지정학적으로 한국에 매우 불리 |

#### 잣대 5: Policy Tailwind (L1B 전용)

현 한국 정부의 정책 우선순위와 일치 정도. 5년 사이클로 변동.

| 점수 | 기준 |
|------|------|
| 5 | 정부 핵심 산업 정책 (예: 반도체, AI, 방산) + 직접적 보조금·세제 혜택 |
| 4 | 정부 정책 우선순위 진입, 보조금·인프라 지원 |
| 3 | 정부 관심 있으나 구체 지원 미약 |
| 2 | 정책 무관 |
| 1 | 정책 역풍 가능성 |
| 0 | 정책적으로 억제 대상 |

**주의**: Policy Tailwind 는 정권 교체 시 급변. 5년 시계로 분석하지 말고 현 정권 잔여 기간 기준으로만 평가.

### 2.6 채점 합산과 통과 임계

> **2026-05-23 변경**: Tech 잣대 폐기로 인해 잣대 수 축소. 통과 임계치도 비율 유지하며 조정.

**L1A (글로벌)**: 3잣대 합산 0~15점.
- 9점 이상: 통과 (Tier 진입 후보)
- 6~8점: 관찰 (Tier 3 thin)
- 5점 이하: 후보 제외

**L1B (한국)**: 5잣대 합산 0~25점.
- 15점 이상: 통과 (Tier 진입 후보)
- 10~14점: 관찰 (Tier 3 thin)
- 9점 이하: 후보 제외

**A·B·C 시차 매력도**는 별도 채점 (섹션 4에서 정의). 위 잣대는 영역 자체의 매력도, 시차는 진입 타이밍 매력도.

## 3. Confidence 등급 — 모든 점수에 부속

채점은 단독으로 의미 없다. 그 점수의 근거가 얼마나 견고한지 같이 표기한다.

### 3.1 등급 정의

| 등급 | 의미 | 표기 예시 |
|------|------|---------|
| **High** | 복수 1차 소스로 검증. 수치 일치. | "TAM 4점 / High" |
| **Medium** | 1차 소스 1개 또는 2차 소스 다수. 추정 가정 일부 포함. | "TAM 4점 / Medium" |
| **Low** | 추정·간접 신호·단일 출처·정성 판단 비중 큼. | "TAM 4점 / Low" |

### 3.2 1차·2차 소스 정의

**1차 소스**:
- 시장 조사 기관 (Gartner, IDC, IEA, BloombergNEF, McKinsey 등)
- 정부·기관 발표 (SEC, DART, IEA, 한국거래소, FED)
- 기업 공시·실적 발표 (10-K, 10-Q, IR 자료, 컨퍼런스콜 transcript)
- 산업 표준 기구·협회 통계

**2차 소스**:
- Sell-side 애널리스트 리포트
- 산업 전문 매체 (SemiAnalysis, The Information, Fabricated Knowledge 등)
- VC 산업 매체 (TechCrunch, The VC, 벤처스퀘어)
- 위키피디아·기타 종합 정보 매체

**비-소스 (Confidence 산정에 카운트 안 함)**:
- X(트위터) 게시물
- 익명 포럼
- 유튜브 영상 (단, 공식 IR 영상은 1차 소스)
- 본인 직관·경험 (별도로 명시할 수 있으나 Confidence 등급은 Low 이하)

### 3.3 Confidence 등급 산정 규칙

각 잣대 점수마다 다음 규칙으로 등급 자동 산정:

- **High**: 1차 소스 2개 이상 + 수치 ±20% 이내 일치
- **Medium**: 1차 소스 1개 또는 2차 소스 3개 이상
- **Low**: 위 조건 미달

### 3.4 Confidence 가 낮을 때의 처리

- Confidence Low 인 점수는 의사결정 시 본인이 별도 검증
- 시스템은 Low 점수를 *그대로* 받아서 합산하되, 합산 점수 옆에 "(Low confidence 잣대 N개 포함)" 표기
- Low 잣대가 2개 이상이면 영역 매력도 판단 자체에 주의 — 다이제스트에 표시

### 3.5 영역 총점의 Confidence

영역 총점의 Confidence 등급은 가장 낮은 잣대의 등급을 따른다.
- 예: TAM High, Moat High, Capital Medium, Tech Low → 영역 총점 Confidence = Low

이는 의도적으로 보수적 처리. 한 잣대라도 불확실하면 영역 전체를 불확실로 본다.

## 4. A·B·C 시차 평가

영역의 매력도(4잣대 합산)와 별개로, **진입 타이밍의 매력도**를 A·B·C 시차로 평가한다.

핵심 명제: **자본은 들어왔고(A), 실물도 따라오기 시작했는데(B), 시장 가격은 아직 반영 안 된(C) 영역**에 진입 알파가 있다. C가 먼저 반영되면 들어가지 않는다.

### 4.1 A 지표 (Capital inflow) — 자본 유입 진행도

자본이 이 영역에 얼마나 들어왔는가. 5단계.

| 점수 | 상태 |
|------|------|
| 5 | 자본 유입 정점. 더 들어올 여지 적음 |
| 4 | 강한 유입 진행 중. 가속 페이즈 |
| 3 | 유입 시작, 초기 단계 |
| 2 | 의향만 있고 실제 집행 미미 |
| 1 | 거의 없음 |
| 0 | 없음 또는 유출 |

**측정 데이터**:
- 빅테크 CapEx 가이던스 분기 변화
- 정부 정책 자금 집행률
- VC 펀딩 (분기 누적, 전년 동기 대비)
- 관련 ETF AUM 변화 (월간)

### 4.2 B 지표 (Physical buildout) — 실물 변화 진행도

자본이 실제 매출·생산·인프라로 전환된 정도.

| 점수 | 상태 |
|------|------|
| 5 | 실물 변화 정점. 대부분 반영 완료 |
| 4 | 강한 실물 변화 진행 중. 매출 증가 가시 |
| 3 | 일부 실물 변화 (파일럿·초기 양산) |
| 2 | 발표는 있으나 실물 변화 미미 |
| 1 | 거의 없음 |
| 0 | 없음 |

**측정 데이터**:
- 핵심 기업 매출·수주 증가율
- 산업 출하량·생산량 통계
- 신규 시설 착공·가동 (데이터센터 MW, 공장 등)
- 장비 출하량

### 4.3 C 지표 (Price reflection) — 시장 가격 반영 정도

시장이 이 영역의 변화를 가격에 얼마나 반영했는가.

| 점수 | 상태 |
|------|------|
| 5 | 완전 반영 또는 과열 |
| 4 | 상당 부분 반영 |
| 3 | 부분 반영 |
| 2 | 일부 반영 시작 |
| 1 | 거의 미반영 |
| 0 | 미반영 또는 음수 (오히려 빠짐) |

**측정 데이터**:
- 관련 ETF 가격 추이 (1년·3년)
- 핵심 종목 멀티플 (P/E, P/S) vs 자기 history
- 핵심 종목 컨센서스 EPS 상향 폭
- 산업 평균 PEG

### 4.4 시차 매력도 산정

진입 타이밍 매력도 = **A + B − C**

| A + B − C | 카테고리 | 의미 |
|-----------|---------|------|
| 6 이상 | Strong entry | 자본·실물 진행, 가격 미반영. 진입 황금기. |
| 3~5 | Good entry | 진입 매력 있음. |
| 0~2 | Fair | 진입 가능하나 매력 보통. 종목별 valuation gate 더 중요. |
| −1 ~ −3 | Late | 가격이 펀더멘털 앞섬. 조정 시 진입 고려. |
| −4 이하 | Avoid | 과열. 신규 진입 부적합. 보유분 청산 검토. |

### 4.5 시차 평가의 갱신 주기

- A·C 지표: 월간 갱신 (시장 가격·자본 흐름 빠르게 변동)
- B 지표: 분기 갱신 (실물 변화는 분기 실적·통계로 확인)

### 4.6 영역 종합 매력도

영역의 종합 매력도는 **4잣대 합산(영역 매력)** × **A·B·C 시차(타이밍 매력)** 두 축으로 본다.

| | Strong/Good entry | Fair | Late/Avoid |
|---|---|---|---|
| **4잣대 12~20점 (강함)** | Tier 1 후보 | Tier 2 | Tier 3 (대기) |
| **8~11점 (보통)** | Tier 2 | Tier 3 | 후보 제외 |
| **0~7점 (약함)** | Tier 3 | 후보 제외 | 후보 제외 |

→ **강한 영역이 늦게 들어가는 것보다, 보통 영역이 일찍 들어가는 것이 더 매력적일 수 있다.**

## 5. Valuation Gate (종목 단위)

영역 매력도와 별개로, 개별 종목이 매수 매력적인지는 **종목 단위 valuation**으로 판단한다. 4개 게이트로 채점.

영역 채점(섹션 2)이 *산업이 좋은가*를 본다면, Valuation Gate는 *가격이 적정한가*를 본다. 둘은 분리.

### 5.1 Gate 1: Self-history (자기 history 대비)

종목의 현재 멀티플 vs 자기 5년 평균.

| 점수 | 기준 |
|------|------|
| 5 | 현재 멀티플 ≤ 5년 평균 − 1σ (저평가) |
| 4 | 현재 멀티플 ≈ 5년 평균 − 0.5σ ~ −1σ |
| 3 | 5년 평균 ± 0.5σ (정상) |
| 2 | +0.5σ ~ +1σ |
| 1 | +1σ ~ +2σ |
| 0 | +2σ 초과 (과열) |

**대상 멀티플**: P/E, P/S, EV/EBITDA 중 해당 산업에 가장 적합한 1~2개.

**주의**: 사업 구조 큰 변화가 있었던 종목은 history 시작점 재정의. (예: AI 시대 진입 후 NVDA는 2022 이후로 history 시작.)

### 5.2 Gate 2: Peer relative (동종 대비)

같은 밸류체인 위치의 peer median 대비.

| 점수 | 기준 |
|------|------|
| 5 | Peer median 대비 −30% 이하 (디스카운트 사유 검증 필요) |
| 4 | −10% ~ −30% |
| 3 | ±10% (정상) |
| 2 | +10% ~ +30% |
| 1 | +30% ~ +50% |
| 0 | +50% 초과 (프리미엄 정당화 필요) |

**Peer 정의**: 같은 영역의 같은 밸류체인 위치 종목 3개 이상. Peer가 적으면 Confidence Low.

### 5.3 Gate 3: Growth-adjusted (PEG)

PEG = 현재 P/E ÷ 향후 3년 EPS CAGR(%)

| 점수 | 기준 |
|------|------|
| 5 | PEG ≤ 0.7 (매우 매력) |
| 4 | 0.7 ~ 1.0 |
| 3 | 1.0 ~ 1.5 (적정) |
| 2 | 1.5 ~ 2.0 |
| 1 | 2.0 ~ 3.0 |
| 0 | PEG > 3.0 (성장률 대비 비쌈) |

**성장률 출처**: 컨센서스 forward EPS 또는 본인 추정. 본인 추정 시 Confidence Low.

### 5.4 Gate 4: Implied expectation (시장 함의)

★ 가장 중요한 게이트. 현재 시총이 이미 반영한 미래 시나리오 vs 본인 분석.

**산정 방식**:
1. 현재 시총 → 향후 5년 매출·마진 가정 역산 (할인율 8~10% 가정, terminal multiple 15~25배 가정)
2. 산업 분석(섹션 2)에서 본인이 추정한 미래 매출·마진과 비교
3. 둘 간의 갭으로 채점

| 점수 | 기준 |
|------|------|
| 5 | 시장이 본인 분석보다 매우 비관적. 본인 뷰가 맞으면 큰 alpha. |
| 4 | 시장이 약간 비관적. |
| 3 | 시장과 본인 뷰 일치. |
| 2 | 시장이 약간 낙관적. 본인 뷰가 맞아도 alpha 제한. |
| 1 | 시장이 명백히 낙관적. |
| 0 | 시장이 비현실적으로 낙관적. 본인 뷰가 맞으면 short 후보. |

**주의**: 이 게이트는 정성·정량 혼합. Confidence는 거의 항상 Medium 이하. 그래도 **이것만이 진짜 알파의 원천**이므로 반드시 작성.

### 5.5 Gate 통과 룰

4개 게이트 점수 합산 0~20점.

| 합산 점수 | 판정 | 의미 |
|----------|------|------|
| 16+ | Cheap | 적극 매수 후보 |
| 12~15 | Fair | 매수 가능, 비중 보통 |
| 8~11 | Stretched | 추가 매수 보류, 보유분 유지 가능 |
| 4~7 | Expensive | 신규 매수 부적합, 보유분 비중 축소 검토 |
| 0~3 | Very expensive | 청산 검토 |

**Gate 4 단독 룰**: Gate 4가 0~1점이면 다른 게이트 점수 무관하게 **신규 매수 중단**. 시장이 비현실적으로 낙관적인 종목은 어떤 멀티플 핑계로도 사지 않는다.

### 5.6 Valuation Gate의 한계 (명시적 인식)

- Valuation Gate는 **사지 말아야 할 종목을 거르는 데** 더 효과적이다.
- Cheap 점수가 매수 신호로 항상 작동하지는 않는다 (value trap 가능).
- 매수 의사결정은 **산업 매력 + 시차 매력 + Valuation Gate** 셋 다 만족해야 합리적.
- Gate 점수가 낮아도 본인 판단으로 매수 가능. 시스템이 매수 결정을 막지는 않는다.

### 5.7 갱신 주기

- 워치리스트 등재 시: 최초 4 게이트 채점
- 분기: 모든 워치리스트 종목 재채점
- 이벤트 발생 시 (대형 실적 변동, 인수합병, 가이던스 급변): 임시 재채점

## 6. 3-Tier 구조와 승강 규칙

영역은 분석 깊이에 따라 3개 tier로 분류.

### 6.1 Tier 정의

| Tier | 깊이 | 영역 수 | 갱신 주기 | 자본 투입 |
|------|------|--------|----------|----------|
| **Tier 1 (Deep)** | 풀 분석. 잣대·A·B·C·종목 매핑·Valuation Gate 모두. 하위 세분화 분해. | 5~7개 | 분기 | 핵심 |
| **Tier 2 (Medium)** | 1페이지 분석. 잣대 + 핵심 종목 매핑까지. 하위 세분화 일부. | 8~12개 | 반기 | 일부 |
| **Tier 3 (Thin)** | 영역 이름·한 줄 요약·핵심 종목 후보 1~3개. | 10~20개 | 연 1회 또는 이벤트 | 안 함 (예외: Theme 영역의 위성 자산) |

### 6.2 승강 규칙

> 점수 임계치는 Tech 잣대 폐기에 맞춰 조정 (2026-05-23).
> L1A는 3잣대 0~15점, L1B는 5잣대 0~25점.

**Tier 3 → Tier 2** (다음 중 하나 이상 충족):
- 잣대 합산 9점 이상 (L1A) 또는 15점 이상 (L1B)
- Capital 잣대 단독 4점 이상 (자본 폭발 시그널)
- 본인이 VC 검토에서 의미 있는 신호 포착 (컴플라이언스 룰 준수)
- 분기 강제 평가 시 매력도 상승

**Tier 2 → Tier 1** (다음 중 둘 이상 충족):
- 잣대 합산 12점 이상 (L1A) 또는 18점 이상 (L1B)
- A·B 진행 중이고 C는 아직 (A+B−C ≥ 3, 시차 매력 큼)
- 본인이 실제 매수 의사 표명
- 핵심 종목 1개 이상 워치리스트 등재 + 활발한 모니터링

**Tier 1 → Tier 2** (다음 중 하나라도 발생):
- 잣대 합산 3점 이상 하락
- A·B·C 모두 가격에 반영 완료 (시차 매력도 −4 이하)
- 기각 조건 발동 또는 근접
- 매매 청산 후 본인 의사로 강등

**Tier 2 → Tier 3** (강등):
- 잣대 합산 6점 이하 (L1A) 또는 9점 이하 (L1B) 로 하락
- 2분기 연속 변화 없음 (관성 청소)

### 6.3 승강 시점

- 정기: 분기 첫 주 (Tier 1·2 재평가), 반기 첫 주 (Tier 3 → 2 승급 검토)
- 비정기: P1 이벤트 발생 시 즉시 재검토

### 6.4 Tier 1 영역 수 제한

Tier 1은 **최대 7개**. 7개 초과 시 가장 낮은 점수의 영역을 강등.

이유: Tier 1은 분기 풀 deep dive 대상. 7개 초과 시 본인 시간 부담 폭발. 강제 캐핑.

## 7. 시계 분류 (Cycle / Core / Theme)

모든 영역은 다음 3개 시계 중 하나로 분류.

### 7.1 분류 정의

| 카테고리 | 시계 | 대표 영역 예시 | 매매 대상 여부 |
|---------|------|--------------|--------------|
| **Cycle** | 1~1.5년 | K-방산, 조선, 정책 사이클, 메모리 사이클 | 매매 대상 |
| **Core** | 1.5~2.5년 | AI 인프라, 광인터커넥트, 전력반도체, 자율주행, GLP-1 | 매매 대상 |
| **Theme** | 시계 무관 (학습) | 핵융합, SMR, 양자, 노화의학 | 매매 비대상 (위성 자산 ≤5% 한정) |

### 7.2 Cycle 영역의 특성

- 사이클이 명확히 시작·정점·종료 단계로 진행
- 정점 도달 후 빠지는 것이 명확해야 함
- 진입은 사이클 초기 (B 지표 막 진행), 청산은 정점 직후 (C 지표 ≥ 4)
- Theme 발화로 인한 단기 급등은 Cycle 영역이 아님

### 7.3 Core 영역의 특성

- 구조적 변화. 1.5~2.5년 안에 A·B·C 시차가 펼쳐짐
- 4잣대 중 Tech(비가역) 점수 3+ 가 일반적
- 정점 도달은 5년 이상 걸릴 수 있으나, 진입·청산 시점은 1.5~2.5년 안에 결판

### 7.4 Theme 영역의 특성

- 1.5~2.5년 안에 매출·이익으로 전환되지 않음
- B 지표가 5년 이상 멀리 있음
- 단기 테마 발화는 자주 일어남 (정책 발표, 빅테크 투자, 실험 결과)
- **본인은 테마 발화 사이클을 타지 않음**. 학습·이해만.
- 위성 자산으로 일부 보유 가능하나 전체 자본의 5% 이하

### 7.5 Theme 영역의 시스템 처리

- Tier 3 thin 으로만 분류. Tier 2·1 승급 불가.
- 단, 영역 안의 핵심 종목 1~3개는 워치리스트 등재 가능 (학습 목적)
- 다이제스트에는 별도 섹션 "Theme tracking" 으로 분리. Core·Cycle 의 우선순위와 섞이지 않음.
- 만약 Theme 영역의 B 지표가 3+ 로 올라오면 (실물 변화 가시화), Core 로 재분류 검토.

### 7.6 시계 분류의 갱신

- 영역의 시계 분류는 연 1회 (1분기 첫 주) 재검토
- 비정기: 영역의 B 지표 큰 변화 시 (Theme → Core, Core → Cycle)

- **일·주간**: 자동 진행, 본인은 복귀 후 누적 다이제스트로 확인
- **월간**: 자동 진행, 본인 검토 보류 시 `pending-review/monthly-YYYY-MM.md` 로 이동
- **분기·반기·연**: 시스템이 초안 작성 후 `pending-review/quarterly-YYYY-QN.md` 로 보류. 다음 정기 시점 전에만 처리되면 됨. 안 되면 다음 분기와 합쳐서 처리.

### 8.3 비정기 갱신 (이벤트 트리거)

- P1 이벤트 발생 (섹션 10에서 정의)
- 워치리스트 종목 분기 실적 발표
- 워치리스트 종목 가이던스 변경
- 영역 내 대형 M&A
- 영역 관련 정책 변경
- 영역 관련 대형 VC 라운드 (시리즈 C+ 또는 1B$+ 밸류)

비정기 갱신 결과는 해당일 다이제스트에 반영 + 변동 큰 경우 별도 알림.

## 10. 이벤트 채점과 다이제스트 통합

### 10.1 이벤트 우선순위 (P1~P5)

들어오는 모든 이벤트(뉴스·공시·수급·가격·VC 라운드 등)는 P1~P5로 채점.

| 점수 | 의미 | 예시 | 처리 |
|------|------|------|------|
| **P1** | 테제 자체가 바뀔 수 있는 사건 | 핵심 공급사 capa 증설/감축, 정책 급변, 신기술 상용화 발표 | 즉시 알림 + 다이제스트 최상단 + 영역 재평가 트리거 |
| **P2** | 테제는 유지되나 가중치·타이밍 조정 필요 | 주요 기업 가이던스 변경, 대형 M&A, 시리즈 C+ 1B$+ 라운드 | 알림 + 다이제스트 상단 |
| **P3** | 워치리스트 종목의 실적·수주 | 분기 실적, 신규 수주 공시 | 주간 다이제스트 포함 |
| **P4** | 산업 동향 참고용 | 시장 리서치 보고서, 컨퍼런스 발표 | 월간 누적 검토 |
| **P5** | 노이즈 | 단순 가격 변동, 일반적 시황 기사 | Raw log 적재만, 다이제스트 미포함 |

### 10.2 이벤트 채점 자동화 룰

각 이벤트의 우선순위는 다음 룰로 자동 채점 (Claude가 적용):

- **P1 조건** (다음 중 하나라도): 워치리스트 종목의 capa 발표, 영역의 정책 급변, 영역의 기각 조건 근접, 핵심 공급사 변화
- **P2 조건**: 워치리스트 종목 가이던스 변경, 영역 내 M&A, 시리즈 C+ 1B$+ 라운드
- **P3 조건**: 워치리스트 종목 분기 실적·수주, Tier 1·2 영역 관련 뉴스
- **P4 조건**: Tier 2·3 영역 관련 일반 뉴스
- **P5 조건**: 위 모두 해당 안 됨

채점 불확실 시 보수적으로 한 단계 낮춤 (예: P2 vs P3 중간이면 P3).

### 10.3 다이제스트 구조

#### 일간 다이제스트 (digests/YYYY-MM-DD.md)

```
1. 🚨 P1~P2 이벤트 (있을 때만, 없으면 "해당 없음")
2. 📊 Tier 1 영역별 임팩트 요약
3. 📈 워치리스트 종목 변동 (P3, 표 형식)
4. 🔍 매크로 변동 1~2줄 요약
5. 🎯 Theme tracking (별도 섹션, Theme 영역 동향)
6. 📅 향후 7일 모니터링 포인트
7. 📝 사용자 확인 필요 (액션 아이템)
8. 🗒️ Raw log (P4~P5, 참조용)
```

#### 주간 다이제스트 (digests/weekly/YYYY-WW.md)

```
1. 한 주 P1~P2 종합
2. Tier 1·2 영역별 가설 점검 (기각 조건 근접 여부)
3. ETF 자금 흐름 주간 변동
4. 매크로 트래커 주간 요약
5. Valuation Gate 변화 (Fair → Stretched 등 전환)
6. 다음 주 모니터링 포인트
7. 액션 아이템
```

#### 누적 다이제스트 (본인 부재 시)

본인이 N일 만에 복귀 시 자동 생성. 형식:
```
1. 부재 기간: YYYY-MM-DD ~ YYYY-MM-DD (N일)
2. 그동안 발생한 P1~P2 이벤트 전체
3. Tier 1·2 영역 점수 변화 요약
4. 워치리스트 종목 가격·수급 변화 (시작·종료 시점 비교)
5. Pending Review 큐 (응답 안 한 알림들)
6. 매크로 주요 변화
7. 액션 아이템 우선순위
```

### 10.4 알림 규칙

- **P1**: 즉시 알림 (텔레그램 봇 또는 이메일)
- **P2**: 일간 다이제스트 생성 시 별도 알림 헤더
- **P3 이하**: 알림 없음, 다이제스트에만 반영

본인 부재 시 P1·P2 알림은 자동으로 `pending-review/` 큐에 누적.

## 11. 임계치 변경과 메타 룰

### 11.1 임계치 변경 규칙

- **변경 가능 시점**: 분기 첫 주에만. 매년 1월 첫 주는 대규모 조정 가능.
- **변경 금지 시점**: 분기 첫 주 외 (특히 채점 진행 중)
- **변경 사유 기록**: 본 문서 하단 업데이트 로그에 반드시 기록 (변경 전·후 수치, 사유)
- **변경 영향 분석**: 임계치 변경 시 과거 1년 채점에 소급 적용해서 영향 분석 (선택)

이유: 채점 중간에 임계치 바꾸면 본인이 본인을 속이게 됨. "이번에 점수가 안 나오네, 임계치 낮추자" 식의 자기기만 방지.

### 11.2 룰 변경 vs 임계치 변경

- **임계치 변경**: 분기 첫 주만. 수치만 바꿈. 예: TAM 5점 임계를 ×10에서 ×8로.
- **룰 변경**: 임계치를 넘어 구조 자체를 바꿈 (예: 잣대 추가, Tier 구조 변경, 가중치 도입). 연 1회 1월 첫 주에만 가능.

### 11.3 채점 일관성 확인

매년 1월 첫 주 회고 시:
- 같은 영역을 6개월·12개월 전 채점과 비교
- 큰 변화가 있다면 (±3점 이상) 사유 검토
- 영역 자체 변화인가, 본인 채점 기준 drift 인가 구분

### 11.4 시스템 자체의 메타 평가

연 1회, 시스템이 본인 의사결정에 실제로 기여했는지 평가:

- 시스템이 추천한 영역 vs 본인 실제 매수 영역의 일치도
- Valuation Gate 가 매수 부적합 판정한 종목 중 본인이 매수한 것의 사후 성과
- 시간 부담 vs 시스템 가치 (지속 가능한가)

평가 결과 따라 시스템 단순화·확장·폐기 결정.

## 12. 컴플라이언스 보완

### 12.1 본인 직접 채널 정보의 처리

본인이 VC 업무에서 알게 된 정보는 다음 룰로 처리:

| 정보 종류 | 시스템 입력 | 매매 활용 |
|----------|----------|---------|
| 산업 트렌드 (예: "이 영역에 시리즈 C 라운드가 증가") | 가능 | 가능 |
| 영역별 펀딩 집계 (공개 데이터 기반) | 가능 | 가능 |
| 검토한 비상장사의 *공개된* 정보 | 가능 | 가능 |
| 검토한 비상장사의 *비공개* 매출·고객·기술 정보 | **금지** | **금지** |
| 검토 중 비상장사의 *비공개* 정보 기반 추론 (그 회사의 상장 고객·공급사 영향) | **금지** | **금지** |

판단 애매하면 입력·매매 둘 다 안 함.

### 12.2 본인 회사 컴플 룰 우선

본인 회사의 임직원 매매 룰 (블랙리스트, 사전 신고, 보유 기간 등) 은 시스템보다 우선. 시스템은 본인 입력만 처리하므로 본인이 자체 판단.

### 12.3 시스템 자체의 컴플 위치

- 시스템은 본인이 입력한 정보를 그대로 처리. 비공개 정보인지 판단하지 않음.
- 시스템 사용 자체가 본인 회사 룰을 위반하지 않는지 본인이 확인.
- 본인이 다른 사람과 시스템 공유 시: 본인 회사 비공개 정보 포함 여부 본인이 검토.
